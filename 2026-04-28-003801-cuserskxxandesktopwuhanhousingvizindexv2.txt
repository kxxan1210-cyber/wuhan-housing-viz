  第五章 房价预测系统设计与实现

  5.1 系统整体架构分析

  5.1.1 系统设计目标

  本系统以武汉市商品住宅为研究对象，融合多元线性回归（MLR）、Lasso
  回归及地理加权回归（GWR）三种预测模型，构建了一套集空间可视化、交互预测与多维
  数据查询于一体的前端单页应用（SPA）。系统的核心目标在于将统计模型的预测结果以
  直观的地图交互形式呈现，使用户能够以自定义参数在任意地理位置完成房价估算，同时
  支持区域横向对比分析，为购房决策及学术研究提供辅助支撑。

  5.1.2 总体架构设计

  系统采用纯前端单页应用架构，无须服务器端运行环境，所有计算逻辑与渲染工作均在浏
  览器本地完成。整体架构可划分为数据层、逻辑层与视图层三个层次，如图5-1所示。

  （1）数据层

  数据层由两部分构成：静态资源数据与动态生成数据。静态资源数据以 JSON
  文件形式存储，包括 gwr_coefficients.json 文件，其中封装了 GWR
  模型各采样点的空间局部系数（coefs）、截距（intercept）、光谷辐射度（radiation
  ）、样本经纬度坐标，以及各模型的全局参数（MLR/Lasso
  系数与截距、特征均值、标准化参数 scaler_mean 与 scaler_scale、特征排列顺序 fea
  ture_order）。动态生成数据指查询模块中依据线性同余伪随机数生成器（种子固定为
  20240427）构建的 280 条模拟房产样本记录，用于数据查询与筛选演示。

  （2）逻辑层

  逻辑层是系统的核心，由以下四个功能模块组成：

  - 空间匹配模块：基于欧式距离最近邻算法，将用户在地图上点击的任意坐标与已知 GWR
   样本点进行匹配，获取该位置对应的局部系数；
  - 预测计算模块：接收用户输入的 6 个特征值（到光谷核心区距离、容积率、绿化率、
  物业费、周边学校数量、周边医疗数量），结合 JSON 文件中的均值和标准化参数构建完
  整特征向量，按照各模型参数完成标准化空间下的线性预测，最终还原为实际房价；
  - 图表渲染模块：基于 ECharts 5
  对区域房价排行、建成年代分布、多点价格对比柱状图及 GWR
  局部系数雷达图进行渲染与更新；
  - 数据查询模块：对模拟房产数据实现多字段组合筛选、多列排序及分页展示。

  （3）视图层

  视图层负责界面布局与交互响应，由左侧固定侧边栏和右侧主内容区两大区域构成。侧边
  栏承载模型选择、功能导航及参数输入等控件；主内容区根据当前导航状态，在地图视图
  、对比分析面板和数据查询表格之间动态切换。地图渲染采用 Leaflet.js
  4，底图调用高德地图瓦片服务。

  5.1.3 关键技术选型

  5.2 系统功能模块分析

  系统功能分为三大主模块：地图预测模块、房价对比模块与数据查询模块，通过顶部导航
  标签页切换。以下逐一分析各模块的设计思路与实现细节。

  5.2.1 地图预测模块

  地图预测模块是系统的核心功能，集成了地图交互、模型切换、空间匹配与价格计算四项
  子功能。

  （1）地图交互与采样点渲染

  系统加载 gwr_coefficients.json
  后，以圆形标记（L.circleMarker）在地图上渲染所有 GWR
  采样点，标记颜色按截距值在蓝色（#1a6abf）到橙色（#ff7800）之间线性插值，颜色深
  浅直观反映各小区基准价格水平。用户悬停标记可查看小区编号及基准价格。地图底图采
  用高德标准矢量瓦片，初始视角定位于武汉市中心（30.5928°N,
  114.3055°E），初始缩放级别为 11 级。

  （2）热力图叠加

  系统支持可选叠加光谷辐射度热力图。热力图强度值源自 GWR 数据中各采样点的
  radiation 字段，代表以光谷广场为核心的高斯距离衰减辐射强度。开启后使用
  L.heatLayer 以五色渐变色阶（蓝—浅蓝—黄—橙—红）染色，直观展示光谷经济辐射范围对
  周边房价的梯度影响，为空间异质性分析提供补充视角。

  （3）空间最近邻匹配

  用户在地图任意位置单击后，系统遍历全部采样点，以 Leaflet 内置的 distanceTo
  方法计算点击坐标与各采样点的球面距离，取最小值对应的点作为预测基准，并在界面上
  显示匹配小区编号及匹配距离。该算法时间复杂度为
  O(n)，在当前数据量级（数百至数千点）下可实时响应。

  （4）多模型预测计算

  完成基准点选取后，用户可通过滑块或数字输入框调整 6 项特征参数（各参数取值范围
  如表5-1所示），点击"执行预测"按钮触发计算。预测流程如下：

  步骤一：构建完整特征向量。 用户直接输入的 6 个字段取实际值，模型中其余特征（如
   log_离学校距离、log_离交通距离、学区_距离交互）按各自的变换规则以数据集均值填
  充。

  步骤二：特征标准化。 按 JSON 文件中记录的训练集均值（scaler_mean）与标准差（sc
  aler_scale）对完整特征向量进行 Z-Score 标准化，保证输入与训练阶段的分布一致：

  $$x_i^* = \frac{x_i - \mu_i}{\sigma_i}$$

  步骤三：模型预测。 三种模型的预测方式如下：

  - GWR 模型：采用当前基准点的空间局部截距与局部系数，计算标准化特征向量与局部系
  数的加权内积，加和截距得到预测价格：

  $$\hat{y}{GWR} = \hat{\alpha}(u_i, v_i) + \sum{k=1}^{p} \hat{\beta}_k(u_i,
  v_i) \cdot x_k^*$$

  - MLR / Lasso 模型：采用 JSON 文件中存储的全局系数（mlr_coef /
  lasso_coef）与截距（mlr_intercept /
  lasso_intercept），对同一标准化特征向量进行全局线性预测，不受地理位置影响：

  $$\hat{y}_{MLR/Lasso} = \hat{\alpha}0 + \sum{k=1}^{p} \hat{\beta}_k \cdot
  x_k^*$$

  预测结果取自然数四舍五入，并附带±10%的参考价格区间，预测价格不低于零元（负值截
  断为零）。三种模型的精度指标如表5-2所示：

  表5-2 三种模型性能对比

  ┌───────┬────────┬───────────────┐
  │ 模型  │   R²   │ RMSE（元/㎡） │
  ├───────┼────────┼───────────────┤
  │ MLR   │ 0.5005 │ 4421.64       │
  ├───────┼────────┼───────────────┤
  │ Lasso │ 0.5038 │ 4407.29       │
  ├───────┼────────┼───────────────┤
  │ GWR   │ 0.6322 │ 3909.86       │
  └───────┴────────┴───────────────┘

  GWR 模型因引入空间非平稳性，R² 较 MLR 提升约 13 个百分点，RMSE 降低约 512
  元/㎡，预测精度最优。

  5.2.2 房价对比模块

  房价对比模块提供两类对比分析功能：基于统计数据的全市概览分析与基于用户自选点的
  多点专题对比。

  （1）全市概览图表

  该子模块内置武汉市 14 个区域的均价统计数据，通过 ECharts
  柱状图按降序排列展示各区房价差异。武昌、江岸、江汉位列均价前三，均超 14000
  元/㎡，而新洲、汉南等远郊区域均价不足 6000
  元/㎡，区域分化显著。同时以环形饼图展示 280 条样本中 2000 年前、2000—2004
  年、2005—2009 年、2010—2014 年、2015—2019 年及 2020
  年后各建成年代的套数分布，反映武汉住宅存量的历史积累特征。

  （2）多点专题对比

  用户可在地图上最多选取 3 个对比点，每个对比点通过最近邻匹配绑定一个已知 GWR
  采样点。各对比点以不同颜色（红/蓝/绿）标注在地图上。选取 2
  个及以上对比点后，系统渲染以下可视化内容：

  - 三模型价格对比柱状图：同一横轴显示 MLR、Lasso、GWR
  三种模型对各对比点的预测价格，便于横向比较模型间的预测差异及各点的价格水平；
  - GWR 局部系数雷达图：从所有 GWR 系数字段中，按绝对值均值筛选前 6
  个最重要特征，以各对比点的归一化系数绘制雷达图，直观展示同一特征在不同地理位置
  的影响方向与强度差异，体现 GWR 的空间异质性；
  - 属性对比明细表：以表格形式列出各对比点的 MLR/Lasso/GWR 预测价格、GWR
  截距、光谷辐射度、坐标，以及前 8 个 GWR
  局部系数，正值以蓝色显示，负值以红色显示。

  5.2.3 数据查询模块

  数据查询模块提供对 280
  条房产样本记录的结构化检索功能，支持多字段联合筛选、多维度排序与分页浏览。

  （1）数据字段设计

  每条记录包含 11 个字段：小区名称、所属区域、开发商、单价、建成年份、主力户型、
  容积率、绿化率、物业费、周边学校数量、周边医疗数量，涵盖区位、建筑、配套三类属
  性，与预测模型使用的特征体系保持一致。

  （2）筛选与排序机制

  侧边栏提供 11 类筛选条件，其中文本类（小区名称）采用模糊包含匹配，枚举类（区域
  、开发商、户型）采用精确等值匹配，数值型字段（价格、容积率、绿化率、物业费、学
  校数量、医疗数量）和年份字段均支持上下界区间筛选。所有条件以逻辑"与"关系复合生
  效，查询结果实时更新。用户可点击任意表头列进行升序/降序排序，默认按单价降序排
  列；当前排序字段以箭头符号标注。

  （3）分页展示

  查询结果每页显示 15 条，分页控件支持首页、末页、相邻页及省略号跳转；当前页按钮
  以蓝色突出显示。单价字段按价格区间着色：高于 14000 元/㎡显示红色，8000—14000
  元/㎡显示橙色，低于 8000 元/㎡显示绿色，便于用户快速识别价格梯度。

  5.2.4 模块间协作关系

  三个功能模块共享同一份 GWR
  系数数据（allGwrData）和模型元数据（gwrMeta），通过全局 JavaScript 变量传递状
  态。"房价对比"模块在选点时会临时切换回地图视图，完成点选后自动返回对比面板，实
  现了地图与分析面板之间的无缝衔接。模型切换（MLR/Lasso/GWR）的状态由全局变量
  currentModel 维护，对地图预测和多点对比模块均实时生效。
